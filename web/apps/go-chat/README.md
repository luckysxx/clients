# @clients/app-go-chat

聊天 Web 应用。

当前状态：

- 仅承接聊天主界面与业务路由
- `/login`、`/register`、`/sso` 仅保留为兼容跳转入口，会转发到认证应用
- 通过 `@clients/shared` 读取 `api-gateway` 下发的运行时配置

本地开发：

```sh
pnpm dev:go-chat
```

当前包含的主要能力：

- 聊天页面与路由
- 认证状态与共享请求能力
- 老链接到认证入口的前端转发

认证页面与 SSO 授权页现在位于 `apps/auth`，生产部署名为 `auth-web`。

如果继续扩展，优先保持这些边界清晰：

1. 页面入口说明
2. 本地启动方式
3. 依赖的后端服务
4. 和共享包的关系
